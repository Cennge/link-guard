# 🛡️ LinkGuard

Лёгкое расширение для Chrome (**Manifest V3**), которое в реальном времени защищает
от перехода на фишинговые сайты, маскирующиеся под популярные бренды.

Вся проверка выполняется **локально, без единого сетевого запроса** — никакие данные
о посещаемых сайтах никуда не отправляются.

## Что детектирует

| Вектор атаки | Пример | Как ловим |
|---|---|---|
| **Тайпосквоттинг** | `gooogle.com`, `faceb00k.com`, `microsofy.com` | Расстояние Дамерау-Левенштейна до списка брендов + leet-нормализация (`0→o`, `1→l`, `3→e`…) |
| **Гомографы / IDN-спуфинг** | `pаypаl.com` (кириллическая `а`), `gооgle.com` | Декодирование punycode (`xn--`) → Unicode, детект смешения алфавитов и «скелетизация» по таблице конфузаблов |
| **Комбосквоттинг** (бонус) | `paypal.secure-login.com` | Имя бренда отдельной меткой в чужом домене |

## Установка (режим разработчика)

1. Откройте `chrome://extensions`.
2. Включите **«Режим разработчика»** (Developer mode) справа сверху.
3. **«Загрузить распакованное расширение»** (Load unpacked) → выберите папку `link-guard`.
4. Готово. Иконка 🛡️ появится на панели.

## Как это работает

```
chrome.webNavigation.onBeforeNavigate   ← перехват до загрузки страницы (main frame)
            │
            ▼
   src/detector.js  analyze(url)
     ├─ psl.js          → eTLD+1 и «ядро» домена (google.co.uk → google)
     ├─ punycode.js     → xn-- → Unicode (для IDN/гомографов)
     ├─ confusables.js  → skeleton(): кириллица/греческий/цифры → латиница
     └─ brands.js       → список брендов + белый список легитимных доменов
            │
            ▼ (если опасно)
   chrome.tabs.update → pages/warning.html  ← страница-предупреждение
```

В MV3 нельзя синхронно блокировать запрос без корпоративного `webRequestBlocking`,
поэтому навигация **перехватывается до коммита** и вкладка перенаправляется на
страницу-предупреждение. Пользователь может вернуться назад, перейти на настоящий
сайт или (на свой страх и риск) продолжить — выбор «продолжить» запоминается на
время сессии, чтобы не зациклить редирект.

### Уровни вердикта

- **danger** — точная визуальная подмена (`faceb00k`, `pаypаl`) или любой гомограф. Высокая уверенность.
- **warning** — опечатка по расстоянию правки, смешанные алфавиты без совпадения с брендом, комбосквоттинг. Решает пользователь.
- **safe** — настоящий сайт бренда, его поддомен, IP, localhost или неизвестный бренд.

## Структура

```
manifest.json          манифест MV3
src/
  background.js        service worker: перехват навигации + сообщения
  detector.js          движок детекции (чистые функции)
  brands.js            бренды + белый список доменов
  confusables.js       таблица конфузаблов, skeleton(), mixed-script
  punycode.js          декодер punycode (RFC 3492)
  psl.js               извлечение registrable-домена (eTLD+1)
pages/                 страница-предупреждение (interstitial)
popup/                 попап: вердикт текущего сайта, статистика, тумблеры
icons/                 иконки 16/48/128
test/detector.test.js  тесты (node test/detector.test.js)
```

## Тесты

```bash
node test/detector.test.js
```

21 кейс: легитимные сайты, тайпосквоттинг, гомографы (с реальным punycode),
комбосквоттинг.

## Настройка под себя

- **Добавить бренд** — допишите запись в `src/brands.js` (`label` + список `domains`).
- **Расширить таблицу похожих символов** — `src/confusables.js`.
- **Порог опечаток** — функция `typoThreshold()` в `src/detector.js`.

## Ограничения (осознанные, ради «лёгкости»)

- Список брендов и публичных суффиксов компактный (не полный PSL ~250 КБ). Легко расширяется.
- Эвристика, а не облачная база угроз: возможны редкие ложные срабатывания (например, реальное слово в одной правке от бренда) — поэтому такие случаи помечаются как `warning`, а не `danger`, и пользователь может продолжить.
